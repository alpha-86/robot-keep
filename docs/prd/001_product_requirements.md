## 1. 产品概述

### 1.1 产品名称
robot-keep 坐姿监测助手

### 1.2 产品定位
一款固定位置的家庭儿童坐姿监测设备，通过摄像头识别儿童坐姿，发现不良姿势时实时语音提醒，帮助儿童养成良好坐姿习惯。

### 1.3 核心用户
- **主要用户**：6-12岁儿童（监测对象）
- **决策者**：家长（购买、配置）
- **使用场景**：书桌前阅读、写字、学习

### 1.4 一期目标
验证人体识别 + 坐姿检测核心算法的可行性，为二期小车形态奠定基础。

## 2. 功能需求

### 2.1 核心功能

#### F1: 人员识别
- **描述**：识别摄像头前的人员，判断是谁
- **优先级**：P0（一期必须）
- **详细需求**：
  - 支持注册家庭成员（通过拍照）
  - 存储人脸特征向量到本地SQLite
  - 支持新增、删除、更新家庭成员
  - 支持离线识别
  - 识别准确率目标：>90%（成年人）

#### F2: 坐姿检测
- **描述**：判断当前人员的坐姿是否正确
- **优先级**：P0（一期必须）
- **详细需求**：
  - 捕获摄像头图像
  - 上传至MiniMax视觉API进行分析
  - 判断坐姿是否正确的标准：
    - 头部是否前倾
    - 背部是否挺直
    - 肩膀是否水平
  - 返回坐姿评分（0-100）
  - 如果评分低于阈值（建议70分），触发语音提醒

#### F3: 语音提醒
- **描述**：检测到不良坐姿时，语音提示儿童
- **优先级**：P0（一期必须）
- **详细需求**：
  - 本地TTS语音合成
  - 提醒内容示例：
    - "小明，你的坐姿有点歪，请坐直一点"
    - "注意背部挺直哦"
  - 提醒频率：检测到不良坐姿后立即提醒
  - 避免频繁打扰：同一错误类型，1分钟内最多提醒2次

#### F4: 历史数据
- **描述**：记录每次检测结果，支持查看历史
- **优先级**：P1（一期必须）
- **详细需求**：
  - SQLite本地存储
  - 记录字段：时间戳、人员ID、坐姿评分、是否提醒
  - 支持按日/周查看坐姿评分趋势
  - 数据保留：最近90天

### 2.2 二期扩展功能

#### F5: 移动能力
- 轮式小车平台
- 室内自主导航

#### F6: 家长通知
- 微信/钉钉通知
- 持续不良坐姿报警

#### F7: 多成员管理
- 家庭成员管理界面
- 个性化坐姿报告

### 3.2 数据流

```
摄像头 (30fps)
    │
    ▼ [每3秒采样1帧]
MediaPipe Pose (提取人体关键点)
    │
    ├──▶ [无人] → 休眠
    │
    ▼ [有人]
InsightFace (人脸识别 → 人员ID)
    │
    ▼
MiniMax Vision API (上传图像 + 关键点 → 获取坐姿分析)
    │
    ├──▶ 评分 ≥ 70 → 记录历史(良好)
    │
    └──▶ 评分 < 70 → 触发语音提醒 → 记录历史(提醒)
```

### 3.3 关键接口

#### MiniMax Vision API 调用

```python
# 坐姿分析请求
response = client.messages.create(
    model="MiniMax-M2.7",
    messages=[{
        "role": "user",
        "content": [
            {"type": "image", "data": image_base64},
            {"type": "text", "text": """分析这张图片中人物的坐姿。
关键骨骼点：{keypoints}
请返回JSON格式：
{
    "person_name": "识别出的人名或'unknown'",
    "posture_score": 0-100的评分,
    "issues": ["问题1", "问题2"],
    "suggestion": "纠正建议"
}"""}
        ]
    }]
)
```

### 3.4 数据库Schema

```sql
-- 家庭成员表
CREATE TABLE family_members (
    id INTEGER PRIMARY KEY,
    name TEXT NOT NULL,
    face_encoding BLOB NOT NULL,  -- 128维人脸特征向量
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 坐姿历史表
CREATE TABLE posture_history (
    id INTEGER PRIMARY KEY,
    member_id INTEGER REFERENCES family_members(id),
    posture_score INTEGER NOT NULL,  -- 0-100
    issues TEXT,                      -- JSON数组
    alerted BOOLEAN DEFAULT FALSE,     -- 是否触发提醒
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 7. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| MiniMax API成本超预期 | 高 | 设置API调用频率限制，本地预处理过滤 |
| 儿童人脸识别准确率低 | 中 | 使用儿童优化模型，或降低阈值 |
| 隐私担忧 | 中 | 所有数据本地存储，明确告知家长 |
| 树莓派性能不足 | 低 | 先在开发电脑验证，优化代码 |
