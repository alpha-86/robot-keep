# 001_人体识别与追踪技术调研

## 1. 概述

robot-keep项目需要具备人体识别能力，能够识别并记录家庭成员身份。本文档调研现有开源方案。

## 2. 核心需求

- 人脸检测：从图像/视频流中定位人脸
- 人脸识别：识别每个人是谁（1:N识别）
- 人员追踪：在连续画面中跟踪同一人

## 3. 开源方案对比

### 3.1 face_recognition (基于dlib)

**技术栈：**
- 底层：C++库 dlib（深度学习模型）
- 上层：Python封装
- 预训练模型：ResNet或类似CNN

**能力：**
- 人脸检测：基于HOG特征或CNN
- 人脸关键点：68个特征点
- 人脸编码：128维向量
- 识别准确率：LFW数据集99.38%（欧美人为主）

**优点：**
- API简洁，易上手
- 支持树莓派
- 纯离线运行

**缺点：**
- 对亚洲人脸识别率偏低
- 儿童面孔识别效果待验证
- 依赖dlib，安装较复杂

**GitHub：** https://github.com/ageitgey/face_recognition

---

### 3.2 InsightFace

**技术栈：**
- 底层：MXNet / PyTorch
- 模型：ArcFace, RetinaFace, SCRFD
- 支持移动端部署

**能力：**
- 人脸检测：SCRFD（高精度快速）
- 人脸识别：ArcFace（业界领先）
- 关键点检测：5点或106点
- 支持人脸属性分析（年龄、性别）

**优点：**
- 对亚洲人脸优化
- 支持移动端/嵌入式
- 多种预训练模型可选
- 识别准确率高

**缺点：**
- 安装配置相对复杂
- 需要GPU加速效果更佳

**GitHub：** https://github.com/deepinsight/insightface

---

### 3.3 MediaPipe Face Detection

**技术栈：**
- Google出品的跨平台框架
- TFLite/EdgeTPU适配

**能力：**
- 人脸框检测
- 实时性好
- 支持多脸

**优点：**
- 移动端性能优异
- 易于集成
- 实时性强

**缺点：**
- 仅检测，无识别能力
- 需配合其他方案做识别

---

### 3.4 YOLOv8 Face

**技术栈：**
- YOLOv8目标检测框架
- 支持人脸专用模型

**能力：**
- 人脸检测
- 可扩展到人脸识别

**优点：**
- 目标检测统一框架
- 速度快
- 易于部署

---

## 4. 推荐方案

### 一期方案（验证阶段）

**人脸检测：** MediaPipe Face Detection 或 YOLOv8-Face
- 速度快，实时性好
- 易于嵌入式部署

**人脸识别：** InsightFace (ArcFace)
- 对亚洲人脸优化
- 支持移动端
- 准确率高

### 二期考虑

结合人脸识别 + 身体特征/衣着识别，提高在背面/侧面的识别能力

## 5. 人员数据库设计建议

- 人脸特征向量存储（128/512维）
- 支持新增/删除/更新人员
- 向量检索可使用Faiss加速

## 6. 成本估算

| 方案 | 硬件要求 | 成本 |
|------|---------|------|
| face_recognition | 普通PC/树莓派4 | 低 |
| InsightFace | GPU加速推荐 | 中 |
| MediaPipe | ARM/EdgeTPU | 低 |
