# 001_硬件选型与采购方案

**版本：** v1.0
**日期：** 2026-03-24
**状态：** 正式

---

## 1. 文档目的

本文档为robot-keep项目提供硬件选型参考，综合评估性能、价格、以及与未来二期（移动小车）、三期（复杂感知）需求的匹配程度。

---

## 2. 一期需求回顾

| 模块 | 硬件需求 |
|------|---------|
| 主控 | 运行Python、数据库、MediaPipe、Whisper |
| 摄像头 | USB或CSI接口，1080p，自动对焦 |
| 麦克风 | USB接口，支持语音输入 |
| 扬声器 | 3.5mm或USB输出 |
| 存储 | SQLite数据库存储 |

## 3. 核心硬件选型对比

### 3.1 主控板对比

#### 选型A: Raspberry Pi 5 8GB ⭐推荐

| 指标 | 参数 |
|------|------|
| 处理器 | Broadcom BCM2712, 4核A76 @ 2.4GHz |
| 内存 | 8GB LPDDR4X |
| GPU | VideoCore VI (支持OpenGL ES) |
| 接口 | 2xUSB 3.0, 2xUSB 2.0, HDMI, CSI, DSI, GPIO |
| 价格 | ¥600（散片）/ ¥700（套装） |
| 功耗 | 5V 5A（25W） |
| AI加速 | 可选Google Edge TPU USB |

**优点：**
- 生态最成熟，文档最全
- 8GB内存足够跑所有一期算法
- 可直接安装Raspberry Pi OS
- CSI接口原生支持Pi Camera
- 配件最丰富，价格透明

**缺点：**
- 无内置NPU，AI推理依赖CPU
- Whisper推理较慢（需优化）

**一期匹配度：** ⭐⭐⭐⭐⭐
**二期匹配度：** ⭐⭐⭐⭐（需外接电机驱动）
**三期匹配度：** ⭐⭐⭐（可加Edge TPU加速）

## 6. 完整采购方案

### 6.0 零成本验证方案（¥0）

**说明：** 如果你只是想先验证算法可行性，不花一分钱。**MiniMax API已购入，此方案完全可行。**

| 设备 | 替代方案 | 价格 | 备注 |
|------|---------|------|------|
| 主控 | 现有开发电脑 | ¥0 | Windows/Mac/Linux都行 |
| 摄像头 | 手机摄像头（USB捕获）或自带摄像头 | ¥0 | 测试足够 |
| 麦克风 | 电脑自带麦克风 | ¥0 | - |
| 扬声器 | 电脑自带扬声器 | ¥0 | - |

**验证方式：**
- 在开发电脑上直接运行Python代码
- 用OpenCV捕获手机摄像头或电脑摄像头
- 所有算法本地运行（MediaPipe、InsightFace）
- **MiniMax API云端调用进行坐姿分析**

**API成本参考：**
- MiniMax视觉API：按调用次数计费
- 假设每3秒检测1次，每天使用2小时 ≈ 2400次/天
- 实际坐姿提醒才需要详细分析，可降低调用频率
- **建议：先估算使用频率，控制API成本**

**优点：** 零硬件成本，快速验证算法可行性
**缺点：** 无法产品化，只能验证核心逻辑

**适合：** 立项初期的技术验证阶段
